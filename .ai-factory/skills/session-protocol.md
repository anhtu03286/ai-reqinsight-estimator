# Skill — Session Protocol

Apply to **every session**, every agent platform.

## On start

- [ ] Read `AGENTS.md`
- [ ] Read `.ai-factory/knowledge/INDEX.md`
- [ ] Identified agent role (product / architect / planner / implement / reviewer)
- [ ] Read the corresponding agent file in `.ai-factory/agents/`

## Before changing code or reviewing

- [ ] Does INDEX have lessons with relevant tags? → read the full lesson file (especially `KL-G-001` after pull/rebase)
- [ ] Task (if implementing): `docs/planning/plan/tasks/task-XXX.md` is within scope; **Files** section is clear
- [ ] Just did `pull` / rebase? → do not modify newly pulled code unless it is in **Files** or **Reason + Scope of Impact + Risk** has been documented

## Before ending the session

- [ ] Review FAIL or recurring issue → lesson created + INDEX updated?
- [ ] Lesson ID written in review or task (if applicable)?
- [ ] Task completed: are there any trade-offs, workarounds, or technical choices that would be done differently next time without a note? → Yes: create a lesson; No: end normally

## Do not

- Do not rely on "chat memory" instead of knowledge files
- Do not end after FAIL without writing a lesson (except one-time typos)
